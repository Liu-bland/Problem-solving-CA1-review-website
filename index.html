  • 涉及主题： Topic 8

  • 分析： 第 2 题专门考察了线性查找和二分查找的步骤描述及区别（占 25 分）。

  • 建议： 不要只记定义，要学会像试卷要求的那样，“清晰地画出数组状态和索引位置的变化”。

  • 涉及主题： Topic 1

  • 分析： 占 25 分。虽然是背诵内容，但考察方式很灵活（如结合快递员案例）。

  • 建议： 重点记忆 Development Cycle 的 6 个步骤 和常用解题策略。

  • 涉及主题： Topic 9

  • 分析： 本卷虽然提到了“排序过程中的数组值”，但核心考的是查找。在大纲中，Selection Sort 和 Bubble Sort 占据了很大篇幅，未来的考试极有可能要求你手动模拟这些排序过程。

最后嘱咐： 逻辑（Logic）是这门课的灵魂。Section B 的 Python 编程（Q4）占据了 25 分，哪怕你的语法偶尔有小错，只要逻辑结构（尤其是缩进和循环控制）正确，依然能拿到大部分分数。

    • 频率： 极高。每份卷子的第一大题必然考察解题步骤或策略。

    • 分析： 相比于上一份考卷，这次增加了 Merge Sort，难度有所提升。这说明你不仅要掌握简单的线性查找，还要熟练掌握分治类排序的手动模拟。

    • 频率： 必考。通常以手动模拟步骤（Trace table 或 diagram）的形式出现。

  3. 算法逻辑设计 (Topic 2, 5, 6): 25%

    • 分析： 重点在于 Selections（If-Else） 和 Repetition（Loops） 的综合运用 。流程图和伪代码是考察你逻辑是否严谨的利器。

    • 分析： Q4 考察的是如何将上述所有逻辑转化为 Python 代码 。这次还涉及到了计数逻辑和“寻找最大值/状态”的逻辑（Topic 7 中关于处理数组/列表的知识）。

🎓 针对性建议：

  • 警惕“前提条件”： Q2(b) 是个经典考点。永远记住：二分查找的前提是数组必须已排序。如果你在未排序的数组上直接写二分查找步骤，那这一题就全丢分了。

  • 掌握“分治法”： 既然这份卷子考了 Merge Sort 和 Divide and Conquer，说明老师很看重你处理大规模复杂问题的拆解能力。建议你多练习如何手绘 Merge Sort 的递归拆分与合并过程。

  • 逻辑的一致性： 你的流程图、伪代码和 Python 代码在逻辑上应该是孪生兄弟。如果你在伪代码里用了循环，Python 代码里却忘了写 while 或 for，那说明你的基础还不牢固。

  • 必考术语： Development Cycle 的六个步骤（Analyse, Design, Code, Test, Implement, Maintain）。

  • 必考策略： 区分 Brute Force（穷举所有可能）与 Heuristics（经验法则/启发式）。

📈 频率分析与复习建议

  1. Section A (理论与算法模拟 - 50分):

    • 复习重点： 练习手画排序（尤其是 Merge Sort ）和查找的每一步状态图。

    • 提示： 这一部分如果步骤清晰，很容易拿满分。不要只写结果，要展示过程 。

  2. Section B (综合应用 - 50分):

    • 复习重点： 重点攻克 嵌套循环 (Nested Loops) 和 计数器/累加器 逻辑（例如计算总订单额和统计特定状态的数量 ）。

    • 提示： Python 题 (Q4) 是拉开差距的关键 。注意 缩进 (Indentation) 和 注释 (Comments)，即使代码运行有小错误，良好的逻辑结构也能获得大部分分数 。

总结： 先确保 Topic 1, 2, 5, 6 的分值稳拿（约占总分 70%），然后再突击 Topic 8 & 9 的复杂算法模拟（如 Merge Sort 的分治图）。这样你不仅能稳过，还能冲击高分 A。

1. 程序规划与开发周期 (The Development Cycle)

  • 试卷+题号： Oct 2025 Q1(c)

  • 对应知识点： Topic 1: 3.1 Program Planning and Development Cycle

  • Canvas 原文链接： 3.1. Program Planning and Development Cycle: Problem Solving (2614) (L03)

  • 背诵内容 (English)：

  1. Design a program: This is the first step where you find the problem, choose a language, and write algorithms or pseudocode. Use flowcharts to make the logic clear. 設計程式： 這是第一步，你需要識別問題、選擇語言，並編寫算法或偽代碼。使用流程圖使邏輯更清晰。

  2. Write a program: Turn your algorithms and pseudocode into the actual program. Using high-level languages like Python or Java makes this step easier. 編寫程式： 將你的算法和偽代碼轉化為實際的程式。使用 Python 或 Java 等高級語言會使這一步更容易。

  3. Correct errors: Check for syntax or logic mistakes. Logic errors cause the wrong results. Use a tool called a compiler to find these mistakes in your code. 糾正錯誤： 檢查語法或邏輯錯誤。邏輯錯誤會導致錯誤的結果。使用稱為編譯器的工具來找出程式碼中的這些錯誤。

  4. Test: This is a very important step. Run the program with different tests to check if the logic is correct and the output has no errors. 測試： 這是非常重要的一步。通過不同的測試運行程式，以檢查邏輯是否正確以及輸出是否有誤。

  5. Debug: Find and fix errors that happen while the program is running. Use a debugger to check the code and variables line by line. 調試： 找出並修復程式運行時發生的錯誤。使用調試器逐行檢查程式碼和變量。

4. 问题解决策略：分治法 (Divide and Conquer)

  • 试卷+题号： Apr 2025 Q1(b)

  • 对应知识点： Topic 1: 2. Problem Solving Strategies

  • Canvas 原文链接：2. Problem Solving Strategies: Problem Solving (2614) (L03)

  • 背诵内容 (English)：

  1. A way to solve hard problems by breaking them into smaller parts that are easier to fix. It has three key steps: Divide, Conquer, and Combine.

    一種透過將困難問題分解為較小部分來解決的方法，這些部分更容易修復。它有三個關鍵步驟：分解、解決和合併。

  2. Divide: Break a big, complex problem into smaller and similar pieces. You can keep breaking them down if they are still too hard.

    分解 (Divide)： 將一個大型複雜問題分解為更小且相似的部分。如果它們仍然太難，可以繼續分解。

  3. Conquer: Solve the small pieces and use their answers to build the final solution.

    解決 (Conquer)： 解決這些小部分，並使用它們的答案來構建最終的解決方案。

  4. Combine: Put the results together. You can solve many parts at the same time to save time. It is a very organized and fast method.

    合併 (Combine)：將結果組合在一起。你可以同時解決多個部分以節省時間。這是一種非常有組織且快速的方法。

 

分治法优缺点总结 (Pros and Cons of Divide and Conquer)

  1. Pros - Easier to solve: It is much easier to fix small pieces than one big, complex problem. 優點 - 易於解決： 將大問題分解為小部分，使解決問題比處理單一複雜問題更容易。

  2. Pros - Saves time: You can solve many small parts at the same time (concurrency), which makes the process faster. 優點 - 節省時間： 你可以同時解決多個小部分（併發），這使得整個過程更快。

  3. Pros - Highly structured: This method is very organized and uses clear steps to reach the final answer. 優點 - 結構嚴謹： 這種方法非常有組織，並使用清晰的步驟來達成最終答案。

  4. Cons - Complex process: Breaking a problem down and combining the results can be hard and may need many repetitive steps. 缺點 - 過程複雜： 分解問題並合併結果可能很困難，且可能需要許多重複的步驟。

5. 查找算法对比 (Search Differences: Linear vs. Binary)

  • 试卷+题号： Oct 2025 Q2(c) / Apr 2025 Q2(b)

  • 对应知识点： Topic 8: 1 & 2 & 3 Algorithm Searching

  • Canvas 原文链接：
    1. 1. Sequential Search: Problem Solving (2614) (L03) 2. 2. Sort and Search: Problem Solving (2614) (L03) 3. 3. Binary Search: Problem Solving (2614) (L03)

  • 背诵内容 (English)：

  1. Sequential Search (Topic 8: 1): Also called Linear Search. It is a simple way to find an item by checking every element in a list one by one until it finds the target or reaches the end.

    順序查找 (主題 8: 1)： 也稱為線性查找。這是一種簡單的查找方法，透過逐一檢查列表中的每個元素，直到找到目標或到達末尾。

  2. Sort and Search (Topic 8: 2): Sorting puts data in order (like 1, 2, 3) to make finding items easier. Searching is faster on sorted data. In Python, you can use the sorted() function to organize a list.

    排序與查找 (主題 8: 2)： 排序按順序（如 1, 2, 3）排列數據，使查找項目更容易。在已排序的數據上查找速度更快。在 Python 中，你可以使用 sorted() 函數來整理列表。

  3. Binary Search (Topic 8: 3): This only works on sorted lists. It finds a value by repeatedly cutting the search area in half.

    二分查找 (主題 8: 3)： 這僅適用於已排序的列表。它透過重複將搜索範圍減半來查找數值。

  4. Binary Search Process: Compare the target with the middle item. If it matches, you are done. If the target is smaller, search the lower half. If it is larger, search the upper half. Repeat until found.

    二分查找過程： 將目標與中間項目進行比較。如果匹配，則查找完成。如果目標較小，則搜索下半部分。如果目標較大，則搜索上半部分。重複此步驟直到找到。

6. 理解问题的步骤 (Understanding the Problem)

  • 试卷+题号： Apr 2025 Q1(a)

  • 对应知识点： Topic 1: 3. Understand Problem Solving Steps

  • Canvas 原文链接：3. Understand Problem Solving Steps: Problem Solving (2614) (L03)

  • 背诵内容 (English)：

  1. Understand the problem: Start by identifying the problem from real life. Ask if you have enough information. Draw a diagram or list the known and unknown parts to help you move forward.

    理解問題： 從現實生活中識別問題。詢問自己是否有足夠的資訊。畫出圖表或列出已知和未知的部粉，以幫助你繼續下一步。

  2. Devise a plan: Create a solution plan after you understand the problem. You can choose strategies like "divide and conquer" based on how hard the problem is and your past experience.

    制定計劃： 在理解問題後創建解決方案計劃。你可以根據問題的難易程度和過去的經驗，選擇如「分治法」之類的策略。

  3. Carry out the plan: Follow every step of your plan correctly to reach the goal. In software, choosing the right tools and languages is important because they affect the time and cost.

    執行計劃： 正確地執行計劃中的每一個步驟以達到目標。在軟件開發中，選擇正確的工具和語言很重要，因為它們會影響時間和成本。

  4. Look back: Review your plan and the results. Check if you can save more money or resources. If the result is not what you expected, you should fix or change the plan.

    回顧檢查： 審查你的計劃和結果。檢查是否可以節省更多資金或資源。如果結果不如預期，你應該修復或修改計劃。
