Topic 1 Summary: Problem Solving & Programming Foundations

 

1. What is Problem Solving?

A problem is an obstacle between a goal and the present state , and solving it involves the process of moving from that present state to the desired goal. Identifying the problem accurately is the most critical step, as incorrect definitions can lead to a significant loss of time and capital.

問題是目標與現狀之間的障礙 ，解決問題則是從現狀過渡到理想目標的過程 。準確識別問題是關鍵步驟，因為錯誤的定義會導致時間與資金的重大損失 。

问题是目标与现状之间的障碍 ，解决问题则是从现状过渡到理想目标的过程 。准确识别问题是关键步骤，因为错误的定义会导致时间与资金的重大损失 。

 

1.1. Problem Classification

Problems are classified based on structure: well-defined problems have clear rules and solutions (e.g., math or soccer) , while ill-defined problems are ambiguous and often require division into sub-problems. Furthermore, knowledge-rich problems require domain expertise , whereas knowledge-lean problems can be solved using general instructions and logic.

問題根據結構進行分類：定義良好型問題具有明確的規則與方案（如數學或足球），而定義不良型問題則具有模糊性，通常需要拆解為子問題來處理 。此外，知識豐富型問題需要專業領域知識 ，而知識貧乏型問題則僅憑一般指令與邏輯即可解決 。

问题根据结构进行分类：定义良好型问题具有明确的规则与方案（如数学或足球），而定义不良型问题则具有模糊性，通常需要拆解为子问题来处理 。此外，知识丰富型问题需要专业领域知识 ，而知识贫乏型问题则仅凭一般指令与逻辑即可解决 。

 

2. Problem Solving Strategies

Action plans range from intuitive Brute Force (testing all possibilities) and Trial and Error to more structured methods like Analogy (applying knowledge from other domains) and Divide and Conquer (breaking complex tasks into smaller, simultaneous ones). Heuristics offer quick shortcuts based on experience but do not guarantee an optimal solution every time.

行動方案包括：直觀的暴力破解（測試所有可能性）與嘗試錯誤法 ；更具結構性的方法如類比法（應用其他領域的知識）與分治法（將複雜任務拆解為可同步處理的小型任務）。啟發法則提供基於經驗的快速捷徑，但不保證每次都能得到最佳解 。

行动方案包括：直观的暴力破解（测试所有可能性）与尝试错误法 ；更具结构性的方法如类比法（应用其他领域的知识）与分治法（将复杂任务拆解为可同步处理的小型任务）。启发法则提供基于经验的快速捷径，但不保证每次都能得到最佳解 。

 

3. Understand Problem Solving Steps

Prof. George Polya’s framework involves four key steps: understanding the problem statement , devising a plan using appropriate strategies , carrying out the plan correctly , and looking back to verify results and seek optimization.

George Polya 教授的框架包含四個關鍵步驟：理解問題陳述 、使用適當策略制定計劃 、正確執行計劃 ，以及事後回顧以驗證結果並尋求優化 。

George Polya 教授的框架包含四个关键步骤：理解问题陈述 、使用适当策略制定计划 、正确执行计划 ，以及事后回顾以验证结果并寻求优化 。

 

3.1. Program Planning and Development Cycle

Programming is the process of writing instructions to achieve a goal. This cycle includes designing the logic (using algorithms/pseudocode) , writing the code , correcting errors , testing against routines , and debugging to fix runtime issues using specialized tools.

編程是編寫指令以達成目標的過程 。此週期包括設計邏輯（使用算法/偽代碼）、編寫代碼 、糾正錯誤 、進行例行測試 ，以及使用專業工具進行偵錯以修復運行問題 。

编程是编写指令以达成目标的过程 。此周期包括设计逻辑（使用算法/伪代码）、编写代码 、纠正错误 、进行例行测试 ，以及使用专业工具进行调试以修复运行问题 。

3.2. Programming Language

Languages are divided into three types: Machine language (binary 0s and 1s, fastest execution) , Assembly language (uses mnemonics, moderate speed) , and High-level languages (hardware-independent and user-friendly, e.g., Python/Java) which require compilers for conversion.

語言分為三類：機器語言（二進制 0 和 1，執行速度最快）、匯編語言（使用助記符，速度中等）及高級語言（獨立於硬件且易於使用，如 Python/Java），後者需要編譯器進行轉換 。

语言分为三类：机器语言（二进制 0 和 1，执行速度最快）、汇编语言（使用助记符，速度中等）及高级语言（独立于硬件且易于使用，如 Python/Java），后者需要编译器进行转换 。

 

4. Summary

Effective program development utilizes tools like flowcharts and pseudocode to create blueprints before coding. Utilizing structures such as sequence, selection (decision-making), and repetition (looping) ensures the code is flexible and efficient.

高效的程序開發在編碼前會利用流程圖與偽代碼構建藍圖 。通過順序、選擇（決策）及重複（循環）等結構，確保代碼具備靈活性與高效性 。

高效的程序开发在编码前会利用流程图与伪代码构建蓝图 。通过顺序、选择（决策）及重复（循环）等结构，确保代码具备灵活性与高效性 。

 

 

 

Topic 2 Overview: Algorithm, Flowchart and Pseudocode

1. Programming Tools

Programming tools are essential for software engineers to move from a present state to a goal state by accurately identifying problems and designing a logical foundation before coding. The planning phase defines the program flow using specialized tools, helping to choose the right language (machine, assembly, or high-level) and identify potential loopholes early.

編程工具： 編程工具對軟件工程師至關重要，透過準確識別問題並在編碼前設計邏輯基礎，實現從現狀到目標狀態的轉化。 規劃階段利用專業工具定義程序流程，有助於選擇合適的語言（機器語言、匯編語言或高級語言）並及早發現潛在漏洞。

编程工具： 编程工具对软件工程师至关重要，通过准确识别问题并在编码前设计逻辑基础，实现从现状到目标状态的转化。 规划阶段利用专业工具定义程序流程，有助于选择合适的语言（机器语言、汇编语言或高级语言）并及早发现潜在漏洞。

 

1.1. Algorithms

An algorithm is a finite, unambiguous set of step-by-step instructions (starting with "Start" and ending with "Stop") designed to guide a programmer through a specific task without being tied to a specific syntax. Effective algorithms must be detailed regarding inputs, outputs, and decision-making behavior to ensure consistent, legitimate results.

演算法： 演算法是一組有限且明確的逐步指令（以「開始」起點並以「停止」終結），旨在引導程序員完成特定任務，而不受特定語法的限制。 有效的演算法必須在輸入、輸出及決策行為方面具備詳細描述，以確保獲得一致且合法的結果。

算法： 算法是一组有限且明确的逐步指令（以“开始”起点并以“停止”终结），旨在引导程序员完成特定任务，而不受特定语法的限制。 有效的算法必须在输入、输出及决策行为方面具备详细描述，以确保获得一致且合法的结果。

 

1.2. IPO Charts

The Input-Process-Output (IPO) chart is a systematic tool used to understand problem statements by identifying the necessary input variables, the required processing actions, and the resulting output solutions. These tables use generalized language to build a bridge between the problem understanding and the development of a formal algorithm.

IPO 圖表： 輸入-處理-輸出 (IPO) 圖表是一種系統化工具，透過識別必要的輸入變量、所需的處理動作以及最終的輸出方案，來幫助理解問題陳述。 這些圖表使用通用語言，在理解問題與開發正式演算法之間建立橋樑。

IPO 图表： 输入-处理-输出 (IPO) 图表是一种系统化工具，通过识别必要的输入变量、所需的处理动作以及最终的输出方案，来帮助理解问题陈述。 这些图表使用通用语言，在理解问题与开发正式算法之间建立桥梁。

 

1.3. Pseudocode

Pseudocode is a language-independent tool that imitates actual programming instructions using keywords (like DECLARE, IF, REPEAT) to describe an algorithm's logic. It is easier to modify than actual code and serves as a direct blueprint for final program development, though its lack of standard syntax can be challenging for very large programs.

偽代碼： 偽代碼是一種獨立於語言的工具，利用關鍵字（如 DECLARE、IF、REPEAT）模仿實際編程指令來描述演算法邏輯。 它比實際代碼更易於修改，可作為最終程序開發的直接藍圖，但對於極大型程序而言，缺乏標準語法可能會帶來挑戰。

伪代码： 伪代码是一种独立于语言的工具，利用关键字（如 DECLARE、IF、REPEAT）模仿实际编程指令来描述算法逻辑。 它比实际代码更易于修改，可作为最终程序开发的直接蓝图，但对于极大型程序而言，缺乏标准语法可能会带来挑战。

 

1.4. Flowcharts

A flowchart is a visual diagram that uses standardized shapes (like boxes for processes and diamonds for decisions) and flowlines to represent the sequential logic of an algorithm from top to bottom. It serves as excellent documentation for sharing logic among developers, though creating them manually for massive programs can be time-consuming.

流程圖： 流程圖是一種視覺化圖表，使用標準化形狀（如代表處理的矩形和代表決策的菱形）以及流線，從上到下呈現演算法的順序邏輯。 它是開發者之間共享邏輯的優良文檔，但對於龐大程序而言，手動繪製可能非常耗時。

流程图： 流程图是一种可视化图表，使用标准化形状（如代表处理的矩形和代表决策的菱形）以及流线，从上到下呈现算法的顺序逻辑。 它是开发者之间共享逻辑的优良文档，但对于庞大程序而言，手动绘制可能非常耗时。

 

2. Control Structures

Regardless of the language, modern program logic relies on three fundamental control structures—sequence, selection, and repetition—to manage complex decision-making and flow control.

控制結構： 無論使用何種語言，現代程序邏輯都依賴三種基本的控制結構——順序、選擇與重複——來管理複雜的決策與流程控制。

控制结构： 无论使用何种语言，现代程序逻辑都依赖三种基本的控制结构——顺序、选择与重复——来管理决策与流程控制。

 

2.1. Sequence

Sequence is the default programming mode where statements are executed one after another in a specific order, creating a simple linear logical flow. While straightforward, it cannot handle complex tasks requiring conditional branches or repeated actions.

順序結構： 順序結構是默認的編程模式，語句按特定順序逐一執行，形成簡單的線性邏輯流。 雖然直觀，但它無法處理需要條件分支或重複動作的複雜任務。

顺序结构： 顺序结构是默认的编程模式，语句按特定顺序逐一执行，形成简单的线性逻辑流。 虽然直观，但它无法处理需要条件分支或重复动作的复杂任务。

 

2.2. Selection

Selection structures, such as "If-else" statements, act as flow control mechanisms that alter the program path based on specific variable conditions. This allows the program to choose between alternate paths, enabling dynamic decision-making during execution.

選擇結構： 選擇結構（如 "If-else" 語句）充當流程控制機制，根據特定變量條件改變程序路徑。 這允許程序在備選路徑之間做出選擇，從而在執行過程中實現動態決策。

选择结构： 选择结构（如 "If-else" 语句）充当流程控制机制，根据特定变量条件改变程序路径。 这允许程序在备选路径之间做出选择，从而在执行过程中实现动态决策。

 

2.3. Repetition

Repetition (iteration or looping) allows specific blocks of code to execute multiple times based on a given condition, using structures like "While" or "For" loops. Care must be taken to avoid infinite loops, where statements execute indefinitely without reaching a termination point.

重複結構： 重複結構（迭代或循環）允許特定代碼塊根據給定條件多次執行，常見結構包括 "While" 或 "For" 循環。 必須注意避免無限循環，即語句在未達到終止點的情況下永無止境地執行。

重复结构： 重复结构（迭代或循环）允许特定代码块根据给定条件多次执行，常见结构包括 "While" 或 "For" 循环。 必须注意避免无限循环，即语句在未达到终止点的情况下永无止境地执行。

 

3. Summary

In conclusion, programming is a sequential development process where tools like IPO charts and flowcharts provide a blueprint to identify loopholes before coding starts. By integrating sequence, selection, and repetition, programmers gain the flexibility to handle complex conditional execution and efficient program flow.

總結： 總之，編程是一個順序開發過程，其中 IPO 圖表和流程圖等工具在編碼開始前提供了藍圖以識別漏洞。 透過整合順序、選擇與重複結構，程序員獲得了處理複雜條件執行與高效程序流的靈活性。

总结： 总之，编程是一个顺序开发过程，其中 IPO 图表和流程图等工具在编码开始前提供了蓝图以识别漏洞。 通过整合顺序、选择与重复结构，程序员获得了处理复杂条件执行与高效程序流的灵活性。

 

 

Topic 3 Overview: Data Types and Invariants

1. Invariants

Invariants are logical assumptions or conditions that remain true before and after the execution of specific code segments, such as loops. They are common to all programming languages and serve as critical tools for debugging, ensuring program output accuracy, and achieving error-free programming by asserting state consistency.

不變量： 不變量是邏輯假設或條件，在特定代碼段（如循環）執行前後均保持不變 。它們通用於所有編程語言，是偵錯、確保程序輸出準確性以及通過斷言狀態一致性來實現無錯誤編程的重要工具 。

不变量： 不变量是逻辑假设或条件，在特定代码段（如循环）执行前后均保持不变 。它们通用语所有编程语言，是调试、确保程序输出准确性以及通过断言状态一致性来实现无错误编程的重要工具 。

 

2. Data Types

Data types define the nature of data and how languages behave in typical situations; they are broadly categorized into primitive (non-abstract/system-defined) and non-primitive (abstract/user-defined) types. While different languages may use varied keywords, these fundamental structures provide the necessary flexibility to implement complex real-life logic within a program. 資料類型： 資料類型定義了數據的性質以及語言在典型情況下的行為；它們大致分為原始類型（非抽象/系統定義）和非原始類型（抽象/用戶定義） 。雖然不同的語言可能使用不同的關鍵字，但這些基礎結構為在程序中實現複雜的現實邏輯提供了必要的靈活性 。

数据类型： 数据类型定义了数据的性质以及语言在典型情况下的行为；它们大致分为原始类型（非抽象/系统定义）和非原始类型（抽象/用户定义） 。虽然不同的语言可能使用不同的关键字，但这些基础结构为在程序中实现复杂的现实逻辑提供了必要的灵活性 。

 

2.1. Primitive Data Types

Primitive data types are pre-defined fundamental structures whose behavior cannot be modified by the programmer. They are built into almost all languages and are optimized for faster execution compared to non-primitive types, serving simple operations like basic arithmetic.

原始資料類型： 原始資料類型是預定義的基礎結構，程序員無法修改其行為 。它們內置於幾乎所有語言中，與非原始類型相比，其執行速度經過優化，適用於基本的算術運算等簡單操作 。

原始数据类型： 原始数据类型是预定义的基础结构，程序员无法修改其行为 。它们内置于几乎所有语言中，与非原始类型相比，其执行速度经过优化，适用于基本的算术运算等简单操作 。

 

2.2. Integer

Integers represent positive or negative whole numbers, including zero, but do not support fractional values. Due to their fast execution speed, they are widely used for counting and maintaining program flow in loops.

整數： 整數代表正或負的整數（包括零），但不支持小數值 。由於其執行速度快，被廣泛用於循環中的計數和維持程序流程 。

整数： 整数代表正或负的整数（包括零），但不支持小数值 。由于其执行速度快，被广泛用于循环中的计数和维持程序流程 。

 

2.3. Float

Floats are numeric types used to store fractional or decimal numbers in memory using single or double-precision binary formats. They are essential for systems requiring high processing capacity to handle extremely small or large numbers.

浮點數： 浮點數是用於存儲分數或小數的數值類型，在內存中使用單精度或雙精度二進制格式表示 。對於需要高處理能力以處理極小或極大數值的系統而言，它們至關重要 。

浮点数： 浮点数是用于存储分数或小数的数值类型，在内存中使用单精度或双精度二进制格式表示 。对于需要高处理能力以处理极小或极大数值的系统而言，它们至关重要 。

 

2.4. Boolean

The Boolean data type supports only two logical values: '0' (false) and '1' (true). It is primarily used in conditional statements to control the program's execution flow and develop Boolean logic like 'AND' and 'OR'.

布林值： 布林資料類型僅支持兩個邏輯值：「0」（假）和「1」（真） 。它主要用於條件語句中以控制程序的執行流，並開發如「與」（AND）和「或」（OR）的布林邏輯 。

布尔值： 布尔数据类型仅支持两个逻辑值：“0”（假）和“1”（真） 。它主要用于条件语句中以控制程序的执行流，并开发如“与”（AND）和“或”（OR）的布尔逻辑 。

 

2.5. Character

The character data type stores a single symbol, letter, or number, typically requiring only one byte of memory. In programming, character values are traditionally enclosed in single quotes (e.g., 'A').

字元： 字元資料類型存儲單個符號、字母或數字，通常僅需一個字節的內存 。在編程中，字元值傳統上用單引號括起來（例如 'A'） 。

字符： 字符数据类型存储单个符号、字母或数字，通常仅需一个字节的内存 。在编程中，字符值传统上用单引号括起来（例如 'A'） 。

 

2.6. String

A string is an extension of the character type, functioning as a sequence or array of characters. While characters use single quotes, strings are represented by double quotes and can include text, numbers, and special symbols.

字串： 字串是字元類型的擴展，作為字元的序列或數組運行 。字元使用單引號，而字串則由雙引號表示，並可包含文本、數字和特殊符號 。

字符串： 字符串是字符类型的扩展，作为字符的序列或数组运行 。字符使用单引号，而字符串则由双引号表示，并可包含文本、数字和特殊符号 。

 

3. Abstract Data Types

Abstract Data Types (ADTs) are user-defined structures where the behavior is determined by specific attributes and functions. They facilitate code reuse and compactness, including types like enums, arrays, linked lists, and stacks.

抽象資料類型： 抽象資料類型 (ADT) 是用戶定義的結構，其行為由特定的屬性和函數決定 。它們有助於代碼重用和精簡，包括枚舉、數組、鏈表和堆棧等類型 。

抽象数据类型： 抽象数据类型 (ADT) 是用户定义的结构，其行为由特定的属性和函数决定 。它们有助于代码重用和精简，包括枚举、数组、链表和堆栈等类型 。

 

3.1. Enumerated (Enum)

The Enumerated (Enum) type allows programmers to assign meaningful names to unique integer values for better code readability. It is ideal for storing sets of unique, non-repeating items like days of the week or colors.

枚舉 (Enum)： 枚舉類型允許程序員為唯一的整數值分配具備意義的名稱，以提高代碼的可讀性 。它非常適合存儲一組唯一且不重複的項目，如星期幾或顏色 。

枚举 (Enum)： 枚举类型允许程序员为唯一的整数值分配具备意义的名称，以提高代码的可读性 。它非常适合存储一组唯一且不重复的项目，如星期几或颜色 。

 

3.2. Array

An array is a collection of elements of the same data type stored in a specific order, similar to a mathematical vector. They can be single-dimensional (rows/columns) or multidimensional (matrices), and elements are accessed via index numbers.

數組： 數組是相同資料類型的元素按特定順序排列的集合，類似於數學中的向量 。它們可以是單維（行/列）或多維（矩陣），並通過索引編號訪問其中的元素 。

数组： 数组是相同数据类型的元素按特定顺序排列的集合，类似于数学中的向量 。它们可以是单维（行/列）或多维（矩阵），并通过索引编号访问其中的元素 。

 

4. Summary

Data structures and invariants are integral to programming; while data types provide implementation flexibility, invariants ensure logical correctness during execution. Failure to maintain invariants, such as in sorting algorithms like bubble sort, directly results in incorrect program output.

總結： 數據結構與不變量是編程中不可或缺的部分；資料類型提供了實現的靈活性，而不變量則確保了執行過程中的邏輯正確性 。若未能維持不變量（例如在冒泡排序等算法中），將直接導致程序輸出錯誤 。

总结： 数据结构与不变量是编程中不可或缺的部分；数据类型提供了实现的灵活性，而不变量则确保了执行过程中的逻辑正确性 。若未能维持不变量（例如在冒泡排序等算法中），将直接导致程序输出错误 。
